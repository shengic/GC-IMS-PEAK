# GC-IMS 二維資料找峰專案：工作流、方法論與流程圖

**Version: ver.01 — by Albert Sheng**

本文件彙整自一系列設計討論，作為 GC-IMS（或同類二維風味分析資料）峰偵測模組的架構藍本。 定位：**可校準的儀器（calibratable instrument）**，而非黑盒模型。所有方法選擇以「可解釋、可校準、可嵌入 SaaS」為最高原則。

**輸入模式：影像模式（IMAGE MODE）**。本版假設**僅能取得 2D 彩色熱圖影像（較高解析度版本）**，無法取得原始 `.mea` 數值矩陣。此假設根本性地改寫管線並犧牲部分能力，詳見第 1 節。

狀態：架構設計階段（尚未進入實作）。文件末列出仍待使用者拍板的領域問題。

---

## 0\. 問題定義

| 項目 | 內容 |
| :---- | :---- |
| **輸入** | **2D 彩色熱圖影像**（jet/turbo 類 colormap 渲染）。非原始數值矩陣。 |
| 物理軸 | X 軸 \= 漂移時間 (Drift Time, DT)，標示為 `drift time [RIP relative]`，刻度自 ≈1.00 起（RIP 正規化）。Y 軸 \= 保留時間 (Retention Time, RT)。**注意此顯示慣例 X=DT、Y=RT，與一般 x=RT/y=DT 文字定義相反。** |
| z（強度） | 編碼於像素顏色，需 colormap 反演還原，有損。 |
| 目標 | 自動找出「可行峰（feasible peak）」的位置（RT, DT），並依規則排除不要的（如 plateau、RIP、拖尾帶）。 |
| 終端用途 | 風味分析（VOC 指紋）；對接**獨立的人工峰清單**（非影像標註，見 §1.3）做校準。 |

### 核心難點（決定方法選擇）

1. **峰高低不一** → 不能用絕對高度門檻，需相對「突出度」。  
2. **相鄰峰（仍可分辨兩個極大）** → 需求是兩個都要保留（不合併）。  
3. **共洗脫 / 融合峰（連極大都只剩一個）** → 偵測法失效，原需數學去卷積；**影像模式下此能力嚴重受限（見 §1）**。  
4. **峰形不一（近高斯但不全同）** → 主幹須無模板。  
5. **plateau（平頂）** → 需量化「平坦度」準則剔除。  
6. **【影像模式新增】colormap 反演、飽和、文字遮擋、軸校正** → 見 §1。

## 1\. 影像模式：能力與限制（本版核心變更）

從「渲染影像」找峰，相較「原始矩陣」找峰，是一個**被限縮的問題**。偵測骨架可沿用（突出度/blob/peak\_local\_max 本就是影像演算法），但下列能力在「資料→彩色圖」渲染時即被破壞，非調參可救。

### 1.1 失去或退化的能力

| 問題 | 成因 | 後果 |
| :---- | :---- | :---- |
| **z 需從顏色反推** | 強度經 colormap 映射成 RGB | RGB→z 反演有損、有歧義；jet 亮度非單調、感知不均，不同 z 可能映到相近色 |
| **飽和區高度永久丟失** | RIP 與高峰頂到 colormap 上限（純紅） | 超過上限者全變同一紅，無法區分高低 → **定量能力基本報廢**，僅未飽和區可談相對強度 |
| **去卷積基本失效** | MCR-ALS/PARAFAC2 需真實強度的雙/三線性結構 | colormap 反演+飽和+遮擋的影像不保有此結構 → **共洗脫融合峰無法乾淨拆開** |
| **文字標註遮擋峰** | 紅字 Area/化合物名、虛線框蓋在峰中心 | 偵測前須移除並修補(inpaint)，否則峰中心像素被汙染 |
| **解析度下採樣** | 影像像素數 \<\< 原始矩陣(\~3150×6123) | 相鄰峰可能在影像中合併；次格點精度退化 |
| **軸校正誤差** | 須讀軸刻度做 像素→物理單位 | 新增誤差來源 |
| **僅相對強度** | 通常無 colorbar 對應絕對強度 | 絕對 z 不可還原 |

### 1.2 能力對照

| 能力 | 原始矩陣模式 | 影像模式 |
| :---- | :---- | :---- |
| 峰偵測 / 定位 | ✓ | ✓（本專案目標，可行） |
| 相對強度排序 | ✓ | 部分（僅未飽和區） |
| 定量（峰高∝濃度） | ✓ | ✗（飽和 \+ colormap 非線性） |
| 共洗脫去卷積 | ✓ | ✗ / 嚴重退化 |
| 次格點精度 | ✓ | 退化（顏色量化 \+ 格點粗） |

**結論：影像模式適合「找出可行峰的位置」，放棄定量與去卷積。若後二者變重要，須回原始資料。**

### 1.3 標註「不是」人工答案（重要釐清）

影像上的紅字 Area/化合物標註是**商業軟體的顯示輸出**，實際操作時會被移除，**不可當作校準用的 ground truth**。 理由：

1. 它們是「另一套演算法的輸出」，非真正的人工專業判讀。拿來當標準答案 → 你的程式變成在模仿那套軟體，而非逼近人工（與 §7 A 模式「要複製的是人工」矛盾）。  
2. 實務上標註會被去除，最終影像只剩顏色場 \= 只有**輸入**、沒有**答案**。

**正確的資料配置（輸入與答案分離）**：

- **輸入**：乾淨（無標註）的顏色場影像。  
- **Ground truth**：由使用者/領域專家對乾淨影像（或原始資料）獨立判讀產生的**人工峰位置清單**，作為與影像分離的另一份資料。其來源、格式須明確（至少含每峰 RT、DT；若含相對強度更佳）。

**後果**：單張影像非自足校準樣本。校準需「乾淨影像 \+ 對應的獨立人工峰清單」成對提供。先前「保留標註版供 ground truth」的建議作廢——標註版無校準價值，無須特意保留。

## 2\. 整體架構：兩層分工

| 層 | 子問題 | 方法本質 | 主力演算法 | 影像模式可用性 |
| :---- | :---- | :---- | :---- | :---- |
| **第一層：偵測** | 峰在哪、有幾個 | 找局部極大 / 地形分析 | **Persistent Homology / 突出度** | ✓ 完整可用 |
| **第二層：解析** | 把融合訊號拆成純成分 | 盲分解 / 去卷積 | MCR-ALS / PARAFAC2 | ✗ 影像模式不可用（保留供未來原始資料模式） |

影像模式下系統**以第一層為主體**。第二層列為「原始資料模式才啟用」的未來能力，架構保留接口但不實作。

## 3\. 方法論

### 3.1 主幹：Persistent Homology / 突出度（第一層，影像模式完整適用）

**淹水模型**：曲面從最高處往下淹水。每出現新局部極大即「誕生」一峰（born=峰高）；兩峰集水盆相連時較矮峰「死亡」（die=鞍點高）。

persistence (突出度) \= born − die \= 峰高 − 通往更高峰所必經之最低鞍點高度

| 難點 | 突出度如何解決 |
| :---- | :---- |
| 高低不一 | 與絕對高度脫鉤，矮峰夠突出就留住 |
| 相鄰峰 | 峰間有鞍點，各有獨立突出度，自然算成兩顆 |
| plateau | 平頂無明顯鞍點、突出度低且寬扁，排序自然沉底 |
| 峰形不一 | 無需峰寬/峰形模板，尺度不變 |

**影像模式補充**：突出度作用在反演後的 pseudo-z 面上。即使 colormap 反演不完美，只要「比背景藍高出多少」的相對地形保留，偵測仍穩健——**定位不需完美反演，排序/定量才需要。**

**實作**：`findpeaks`(topology) 或 union-find 自寫。

### 3.2 輔助量

| 輔助量 | 用途 | 弱點 |
| :---- | :---- | :---- |
| **平坦度分數** | 抓 plateau | — |
| **Hessian 曲率** | 交叉確認真尖峰 vs 平頂 | 二階導對雜訊敏感，僅參考 |
| **Blob 偵測 (LoG/DoG/DoH)** | 獨立多尺度交叉驗證 | — |

**平坦度分數**：

頂部尺度 \= 連通區 { z ≥ P − δ\_small } 等效半徑   （δ\_small ≈ 雜訊或峰值 1–2%）

整峰尺度 \= 連通區 { z ≥ P − δ\_large } 等效半徑   （δ\_large ≈ 峰值 50%）

平坦度   \= 頂部尺度 / 整峰尺度

尖峰→趨近 0；plateau→趨近 1。門檻由人工資料校準。

### 3.3 去卷積方法（第二層，影像模式不可用，保留供未來）

影像模式下停用。以下保留供未來「原始資料模式」啟用。

| 方法 | 一句話 | 強項 | 弱點 |
| :---- | :---- | :---- | :---- |
| **MCR-ALS** | D 分解成 C·Sᵀ，交替最小二乘 | 解重疊峰主力；不假設峰形 | 旋轉模糊性（解不唯一） |
| **PARAFAC2** | 多樣本三線性，放寬 RT 對齊 | 抗 RT 漂移；解較唯一 | 計算貴；要規整立方 |
| **SIML** (2024末) | 強化版 MCR-ALS（小波+傅立葉位移校正） | 兼具解析力與抗漂移、抗雜訊 | 僅 GC×GC-TOFMS 驗證 |

共病：成分數(component number)難自動定。三者皆「R²高≠拆對」，需化學先驗約束。

## 4\. 端到端工作流（影像模式）

影像模式在原九步前，新增 I0–I4 影像前處理階段。

【影像前處理階段（影像模式專屬）】

\[I0\] 影像載入與繪圖區裁切  → 載入較高解析度影像；分離繪圖區 vs 軸/邊框/標題

   │

\[I1\] colormap 識別與反演    → 識別 colormap（jet/turbo/vendor）；建 LUT，RGB→pseudo-z

   │                          以 KD-tree 找最近色映射；標記飽和像素（純紅=高度未知/下界）

   │

\[I2\] 標註移除與修補        → 偵測紅字、虛線框；移除並 inpaint(cv2.inpaint)

   │                          修補區峰可信度標低（標註不作 ground truth，見 §1.3）

   │

\[I3\] 軸校正               → 偵測軸刻度位置+數值(OCR或手動)，建 像素↔(RT,DT) 映射

   │

\[I4\] RIP 與拖尾帶遮罩      → 遮罩 RIP 垂直巨柱(X≈1.00)；偵測並排除沿軸延伸的垂直亮帶

   │                          標記飽和區為「高度未知」

【峰偵測階段（沿用原管線，作用於 pseudo-z 面）】

\[3\] 候選極大值初選        → peak\_local\_max 寬鬆門檻；min\_distance；含邊界

   │

\[4\] 對每候選算指標        → 突出度/平坦度/Hessian/邊界/是否飽和/pseudo-z/座標

   │   （只測量，不剔除）     輸出完整候選表

   │

\[5\] 套門檻篩選            → \[第三層參數\] prominence/平坦度/邊界/前N；建議用相對量

   │

\[6\] 次格點精修(退化)      → 局部高斯/拋物面擬合求次格點；影像模式精度有限，標記可信度

   │

\[7\] 與人工結果比對        → 容許半徑 r 配對；命中/漏報/誤報 \+ 吻合分數

   │   （校準引擎）           ▲ 人工答案來源 \= 獨立人工峰清單（非影像標註，見 §1.3）

\[8\] 參數最佳化            → 掃描第5步門檻；校準集/驗證集防過擬合 ─回授至\[5\]─┘

   │

\[9\] 人工覆寫與輸出        → 手動鎖定/排除；輸出峰表(RT,DT,相對強度,可信度…)+視覺化

## 6\. 三層參數結構

| 層 | 階段 | 參數 | 風險 | 調校頻率 |
| :---- | :---- | :---- | :---- | :---- |
| **影像前處理** | I0–I4 | colormap 反演法、飽和門檻、inpaint 半徑、軸校正點、RIP/拖尾遮罩範圍 | **高**（錯誤汙染整個 pseudo-z 面） | 一次定好、嚴格驗證 |
| **第一層** | 前處理 | 平滑 σ | 中 | 少動 |
| **第二層** | 偵測 | min\_distance、初步門檻 | 中 | 偶爾 |
| **第三層** | 篩選裁決 | prominence/平坦度/邊界/前N | **低**（可逆） | **主戰場，頻繁微調** |

**關鍵原則**：

- 自動最佳化（第 8 步）**只動第三層**。影像前處理與第一層高風險決定一次定好。  
- **影像前處理階段是影像模式新增的最高風險層**——colormap 反演、軸校正、遮罩若錯，後續全錯，須以已知參考（如 RIP 已知漂移 8.09ms、軸刻度）嚴格驗證。  
- 參數間有交互作用，不可獨立調 → 用組合掃描 \+ 量化吻合分數自動評比。  
- 門檻盡量用**相對量**（佔樣本最大 pseudo-z 或雜訊量級的百分比）。

## 7\. 校準哲學（A → B 兩階段）

**「校準」非「機器學習訓練」**：演算法固定、參數少(\~5–10)且有物理意義，僅搜尋對人工判讀資料整體最吻合的一組設為預設。優點：可解釋、要的資料少、可重現。

| 模式 | 定義 | 用途 | 誤報處理 |
| :---- | :---- | :---- | :---- |
| **A** | 人工結果當要複製的標準答案 | 開發初期：建立可信基準 | 重罰 |
| **B** | 人工結果當參考，容許找出人漏掉的峰 | A 穩定後：挖掘人工漏標 | 罰則調低/標記待裁決 |

**順序：先 A 後 B**，理由是**可驗證性**（A 有明確成功標準）。**A→B 是評分權重切換，非改寫程式**，底層偵測/指標/篩選不動，可隨時切回 A 驗證。

**防過擬合（必做）**：多份人工判讀資料切「校準集/驗證集」(七三分或 k-fold)；只用校準集搜參數，未參與調參的驗證集量吻合度。A 穩定定義：同一組參數在多份資料達高吻合，剩餘不吻合多為人工判讀差異。跨樣本校準前提：門檻全用相對量。

**校準產物管理（如儀器校準）**：最佳參數存版本化設定檔(JSON)，記錄用哪幾份資料、何時、驗證集吻合度。可重新校準、比較新舊，演算法本體不改。

**回授的兩個層次**：

- **層次一（手動回授）**：程式匯出座標→人工比對→使用者描述錯誤「模式」（漏報峰矮不矮/在不在密集區/誤報在不在 RIP 旁）→ 調參。早期用，建立參數直覺。回報**現象特徵**比指定參數有效（參數間有交互作用，該動哪個由演算法判斷）。  
- **層次二（自動回授）**：比對與評分程式化，參數掃描自動找最佳組（第 8 步）。後期用。先做層次一驗證比對邏輯正確，再固化為層次二。

**彩色比對圖（比對器標準輸出）**：程式依座標比對**自動產生**標記圖（綠=命中、紅=漏報、黃=誤報），每標記附帶該峰指標（突出度/平坦度/是否飽和）。角色是**加速「人」看出錯誤模式**，非供 AI 反推參數。人從圖看出帶特徵的模式（如「漏的都是矮峰」「誤報都擠在某區」）再文字回報——此「帶原因線索的模式」才是可據以調參的資訊。手動圈圖反推參數不可靠（圖示「哪裡錯」但給不出「為何錯」，且目視座標有損）。

## 8\. 比對與評分（第 7 步細節）

校準回路有**兩條獨立的回授路徑**，受不同參數層控制，**不可混為一談**：

| 回授路徑 | 問的問題 | 比對量 | 受哪層參數控制 |
| :---- | :---- | :---- | :---- |
| **偵測回授** | 峰在不在 | 命中/漏報/誤報 | 第三層篩選（prominence/平坦度/min\_distance） |
| **定位回授** | 峰中心在哪 | 配對峰的座標偏移向量 | 影像前處理層（軸校正/colormap）+ 第6步次格點精修 |

### 8.1 偵測回授（命中/漏報/誤報）

- **配對**：自動峰↔人工峰。因 RT/DT 兩軸尺度與單位不同，**用各軸獨立容許窗 (r\_RT, r\_DT) 的矩形，非單一圓形半徑**；或先各軸正規化再用圓形。容許窗與 min\_distance 同尺度。窗太大→虛假高吻合；太小→偏移把對的判成漏掉。  
- **一對一配對規則**：一顆人工峰的窗內有多顆程式峰時，須唯一指派（最近鄰貪婪初期夠用；嚴謹用匈牙利演算法），配掉不再配，避免命中灌水。  
- **三個數**：命中(TP)/漏報(FN)/誤報(FP)。  
- **取捨**：門檻寬→FP增；緊→FN增。不存在兩者皆零。**須先定 FN vs FP 何者更不可接受**（風味分析推測漏真峰比多記雜訊嚴重→偏高召回率，待確認）。  
- **記分板（手動階段也必備）**：每輪都算並記錄總命中/漏報/誤報，當「不退步」的客觀守門員，防手動微調「局部修好、全局退步」。

### 8.2 定位回授（偏移向量分析）

針對已配對成功的峰，分析「程式座標 − 人工座標」的偏移向量集合。**關鍵是看整組分布，非單顆**：

- **系統性偏移（可修正）**：多數配對峰偏移方向/量級一致（如全偏右 0.02 RIP）→ 來自軸校正或 colormap 反演的系統誤差。算平均偏移向量整體補償，一次對準所有峰。**此修正屬影像前處理層，非找峰門檻層。**  
- **隨機散布（不修，硬修=過擬合）**：偏移方向不一致 → 人工目視手抖 \+ 次格點誤差的雜訊。無單一參數可修（動參數是所有峰一起動），應由容許窗吸收，逐顆遷就會在別處製造新偏差。

### 8.3 重要原則：人工是偵測的標準，未必是定位的標準

人工結果對「峰在不在」可靠；對「峰中心精確座標」未必比演算法的高斯擬合質心準（肉眼點中心誤差可能更大）。 **故：偵測信人工（A 模式照舊）；精確定位信擬合，不為貼合人工目視而犧牲真實精度。** 例外：發現系統性偏移時，代表是程式的軸校正/反演有系統錯誤，此時對齊人工是對的（該修的是程式）。

### 8.4 影像模式特例

被 inpaint 修補區、飽和區的峰，比對時應降權或單獨檢視——這些位置的偵測本就較不可信。

| Blob 交叉驗證 | `skimage.feature.blob_*` |  |
| 突出度/持續同調 | `findpeaks`(topology) 或 union-find 自寫 |  |
| 2D 高斯擬合 | `lmfit` 或 `scipy.optimize` | 次格點精修(退化) |
| GC-IMS 參考 | `gc-ims-tools` (v0.1.10) | 原始資料模式才直接相關 |
| **比對/評分邏輯** | — | **自寫（系統大腦）** |
| **人工覆寫機制** | — | **自寫** |
| ~~MCR-ALS / PARAFAC2~~ | ~~`pymcr` / `tensorly`~~ | **影像模式停用**，保留供未來原始資料模式 |

**不建議的方向**（明確排除）：

- **深度學習找峰**：要數百\~數千標註、黑盒、與「可校準儀器」定位衝突。  
- **2D FFT 找峰**：方向錯配（頻率域 vs 空間域）。僅前處理對週期性結構雜訊做陷波濾波有用。  
- **端到端 IMSS+ML**（果汁摻假論文路線）：跳過顯式找峰，黑盒、要大量標註，與本專案價值主張不同。

**部署環境**：RTX 5090 / Windows 11。影像式峰偵測資料量小，純 CPU 可跑。

## 10\. 設計哲學總綱

**把會變的（參數、策略）與不變的（演算法骨架）徹底分開。**

- 影像前處理與峰偵測分離（I0–I4 vs 3–9）  
- 測量與裁決分離（\[4\]只量、\[5\]才篩）  
- 偵測與解析分層（影像模式僅第一層）  
- A/B 靠評分權重切換，非兩套程式  
- 校準參數版本化（如儀器校準紀錄）  
- 通用用輪子、領域決策自寫  
- **影像模式 vs 原始資料模式：同一偵測骨架，前處理層與第二層可插拔**

## 11\. 待使用者拍板的領域問題

| \# | 問題 | 影響 |
| :---- | :---- | :---- |
| 1 | **A 還是 B 起步**（已定：先 A 後 B） | 第7步 FP 是否重罰 |
| 2 | 人工結果記位置，還是位置+強度 | 吻合分數要不要比對強度排序 |
| 3 | 手上有幾份人工判讀資料 | 第8步能否交叉驗證 |
| 4 | FN vs FP 何者更不可接受 | 最佳門檻落點 |
| 5 | plateau 操作型定義（目前僅目視） | 平坦度門檻數值 |
| 6 | 相鄰峰最小可分辨間距 | min\_distance 與比對半徑 r |
| **7** | **【影像模式】colormap 是否已知**（jet?turbo?vendor?）；能否取得 colorbar/圖例 | colormap 反演準確度；能否還原相對強度 |
| **8** | **【影像模式】ground truth 來源**：能否取得乾淨無標註影像（輸入）+ 對應的**獨立人工峰清單**（答案，非影像標註） | §1.3；校準是否成立 |
| **9** | **【影像模式】可取得影像的解析度** | 相鄰峰能否分辨、次格點精度上限 |
| **10** | **【影像模式】是否需區分飽和峰的高度** | 若需 → 影像模式不足，須回原始資料 |
| **11** | **輸出需求**：僅 (RT,DT) 位置？或要相對強度排序？ | 決定是否需精確 colormap 反演 |

**下一步建議**：取得一張較高解析度的**乾淨（無標註）影像**作為輸入，並**另外**取得對應的獨立人工峰清單作為 ground truth（兩者分離，見 §1.3）。先驗證 colormap 反演與軸校正能否以已知參考（RIP=8.09ms、軸刻度）達到可接受誤差；再寫參數化第一版：輸出所有候選峰連同分數（不剔除）+ 比對程式算命中/漏報/誤報 \+ 門檻掃描。 （若暫時無獨立人工清單，可先只做偵測端、輸出候選峰表供目視檢視，校準留待 ground truth 到位。）
